## 🔄 数据迁移

### 重要提示
⚠️ **如果已有生产数据，需要进行数据迁移！**

### 迁移步骤

#### 1. 备份现有数据
```bash
# 在 Vercel 部署前，备份现有数据
# 访问 /api/interns 和 /api/jobs 保存 JSON 数据
```

#### 2. 更新数据结构
为每个实习生记录添加新字段：
```json
{
  "startDate": "随时",
  "baseLocation": "北京",
  "workType": "hybrid",
  "employmentType": "intern"
}
```

为每个职位记录添加新字段：
```json
{
  "baseLocation": "北京",
  "workType": "hybrid",
  "employmentType": "intern"
}
```

#### 3. 恢复数据
```bash
# 部署后，通过 API 重新提交更新后的数据
# 或直接编辑 data/interns.json 和 data/jobs.json
```

### 无数据场景
✅ 如果是全新部署或测试环境，无需迁移，直接部署即可。

## 🐛 常见问题

### Q1: 旧数据无法显示新字段
**原因**: 旧数据缺少新字段
**解决**: 按照数据迁移步骤更新数据

### Q2: 模态框无法打开
**原因**: 可能是浏览器缓存
**解决**: 清除缓存或强制刷新 (Ctrl+Shift+R)

### Q3: 表单提交失败
**原因**: 新字段验证失败
**解决**: 确保所有新字段都已填写

### Q4: 构建失败
**原因**: TypeScript 类型错误
**解决**: 运行 `npm run build` 查看详细错误

## 🔄 回滚计划

### 如果部署出现问题

#### 方法 1: Vercel Dashboard 回滚
1. 访问 Vercel Dashboard
2. 选择项目
3. 点击 "Deployments"
4. 找到上一个稳定版本
5. 点击 "Promote to Production"

#### 方法 2: Git 回滚
```bash
# 回滚到上一个版本
git revert HEAD

# 推送回滚
git push origin main

# Vercel 自动部署回滚版本
```

#### 回滚后的提交
```

3e03191 docs: 添加最终交付报告
a2c4591 docs: 添加项目交付清单
```

回滚到 `3e03191` 即可恢复到 V1 版本。
